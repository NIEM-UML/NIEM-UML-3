Model Package Description

 

There are very few implementations of the MPD Specification.  We have fabricated some sample MPDs by adding MPD Catalog to published IEPDs.   These fabricated MPDs are the basis of the MPD Import action, which may be executed using the following procedure:

 

•      From the Magic Draw Main Menu: FileèImport fromèNIEM MPD.   This will open a file selection dialog to select an mpd zip file.

•      The Sample MPDs may be found at the following location:

o   [magic draw install]/samples/niem/mpd/*.mpd.zip

•      When one of these sample MPDs is selected, transformation from MPD to UML begins.

 

The transformation:

•      Transforms MPD artifacts to the PIM Perspective of NIEM-UML, including the following artifacts:

o   Catalog, which is used to populate the <<ModelPackageDescription>> Component and its relationship to other artifacts.

o   Schemas, which are used to populate <<InformationModel>> Packages,  their owned Classifiers, Properties, etc.

o   XML Sample Instance Documents, which are used to populate the UML Representation for Instance Documents.

•      Converts NIEM 2.0 to 2.1 as required. 

•      Loads NIEM Reference Models and establishes the <<References>> Relationships to MPD subset models

•      Loads NIEM-UML Profiles

•      Loads XML Primitive Types

•      Load NIEM-DSL

•      Creates and lays out Diagrams

 

Importing an IEPD

The procedure for importing a NIEM IEPD:

 

•      From Magic Draw Main Menu: FileèImport fromèNIEM IEPD.   This will open a File Selection Dialog.

•      From the File Selection Dialog, select either a *.zip file or the root folder of an unzipped IEPD

•      A Schema selection dialog is opened:

•      Select a set of exchange schemas.   For well-defined IEPDs, this selection is sufficient to determine the exchange, extension, and subset schemas constituting the IEPD.

•      Select a set of extension schemas.  For ill-formed or incomplete IEPDs, it may be necessary to explicitly select extension schemas.   It is not necessary to select schemas which form the transitive closure of schemas referenced by the exchange schemas.

 

The transformation process:

 

•      Creates a small MPD Catalog which identifies the files selected in the Schema Selection Dialog.

•      The rest of the transformation is the same as described above for importing an MPD.
