Creating An IEPD Project

NIEM Modeling Projects

To create a new IEPD Project, the following procedure may be used:

 

•      From Magic Draw main menu: FileèNew Project…

•      Select NIEM Modeling category

•      The following options are available for IEPD Projects:

o   NIEM IEPD-All Project.   This project will include all NIEM Reference Models.

o   NIEM IEPD-Core Project.  This project will include the NIEM-Core Reference Model, plus all of its pre-requisite models.

•      Supply the Project Name, Project location if desired

•      Click OK

 

 

 

The New Project includes:

o   NIEM Reference Models

o   NIEM-UML Profiles

o   XML Primitive Types

o   NIEM-DSL

o   <<ModelPackageDescription>> Component, configured as an IEPD and with required <<ModelPackageDescriptionFile>> Usages to changelog, xml samples, business-rules

o   Skeletal exchange, extension, subset models to meet minimum NDR/MPD configuration requirements

o   XML Sample Documents, as required by MPD.  These are expected to be replaced by XML Sample Documents based on the modeled exchange <<InformationModel>>.

o   A change log configured for a new IEPD.

o   Business-rules package to manage business constraints and validation.

o   Various skeletal diagrams with some hints on how to start modeling the IEPD.

 

 

The <<ModelPackageDescription>> Component is configured for an iepd, as illustrated in the Class Diagram included with the new project.  It has <<ModelPackageDescriptionFile>> Usages to the initial XML Instance Document, to the business-rules suite, and the change log.  The <<ModelPackageDescription>> Component imports the exchange <<InformationModel>>, which is sufficient to identify the base schema set at the PIM level of modeling abstraction.

 

The skeletal niem-core subset <<InformationModel>> is tagged and configured as a subset of the niem-core Reference Model.
