Analysis

NIEM-UML analysis facilities include:

·         validating compliance to NDR (NIEM Design Rules), MPD (Model Package Description) specification, and other NIEM Specifications,

·         support for constraint models and business rules, including specification of constraints using an enhanced OCL library,

·         active validation:

o   Compliance with NDR, MPD, and other NIEM rules/specifications

o   Compliance with all UML constraints (using the comprehensive eclipse based algorithms implementing OMG UML specified constraints)

o   Compliance with XSD constraints (using eclipse based XSD validation machinery) for models and XML Instance models

o   Compliance with model-specific OCL (using an extended OCL implementation derived from eclipse) as applied to XML Instance models

·         Relation Maps

·         NIEM-UML Derived Properties.  Facilitates understanding and navigation of NIEM-UML concepts, hiding underlying composite UML constructs.  Used to expose NIEM-UML concepts in Specification dialogs, tables, relation maps.

·         Change log.

 

Below are listed some of the topics you might be interested when starting to work with NIEM-UML Analysis:

·         Change log.  A change log is required for every MPD.  NIEM-UML utilizes Magic Draw model differencing technology to construct representations of changes between a base-lined model and current model version.

·         Relation Maps

·         Derived properties

·         Active Validation.

·         Business Rules.   Business Rules are modeled as OCL invariant constraints applied to Classifiers.   The business rules are used to actively validate modeled XML Instance Documents and may be used to provision technology specific constraint implementations.
