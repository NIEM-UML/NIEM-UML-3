Active Validation

Coverage

Active Validation instantly checks the accuracy, completeness, and correctness of a model, displays errors in

the model, and suggests solutions.

NIEM-UML includes active validation for:

·         NDR, MPD, and other NIEM Specification Rules. 

o   Non-computable rules are excludes

o   Rules enforced by MPD Transformation are excluded

o   Rule violations correctable by MPD Transformation are reported with Severity level Info.   Although correctable by transformation, manual adjustment is recommended.

·         Schema validation based on eclipse XSD implementation.  This includes active validation for facet consistency, instance value compliance, restriction constraints, etc.

·         UML validation based on eclipse UML validation algorithms, which covers most OMG UML Constraints.

·         NIEM-UML consistency

Violations are presented in the following forms:

·         Augmented icons in browser trees

·         Color borders on diagrams

·         Tooltips when hovering over element with validation condition

·         The Failure Indicator/count display on status line at bottom of Magic Draw window frame.

·         Active Validation Results Window

Validation actions appear in the contextual menu for elements with validation conditions.  For each validation condition, the following actions may be available:

·         (Optionally) Fix-up action(s) which should remove the validation condition.

·         Show and select the validation condition detail in the Validation Results Window.

The Validation Results Window may also be shown by clicking on the Failure Indicator on the Status Line.

Active validation may be controlled using one of the following procedures:

·         From Magic Draw main menu:  AnalyzeèValidationèActive Validation Options…  This will open the Project Options Dialog

o   In the Project Options Dialog, select “General project options” Category on left pane

o   In right pane, scroll down to “Active Validation” category.

o   Selecting a property will result in  a description for that property appearing at the bottom of the panel.

o   Select the property value field to change the value.

·         The Project Options Dialog may also be opened from Magic Draw main menu: OptionsèProject

 

 

•      Validation enablement and frequency are Environment Options.  The Environment Options Dialog may be opened from the Magic Draw Main Menu:  OptionsèEnvironment

o   On the left pane, select the “General” category.

o   On the right pane, scroll down to the Active Validation category.

o   Selecting a property will result in a description for that property appearing at the bottom of the panel.

o   Select the property value field to change the value.

•      Active Validation may also be enabled/disabled via the toggle button accessed from the Magic Draw Main Menu:  AnalyzeèValidationèEnable Active Validation
