NIEM-UML

·         About NIEM

The National Information Exchnage Model (NIEM) provides a commonly understood way to connect data that improves government decision making for the greater good.

NIEM is a national program supported by the federal government, NIEM connects communities of people who share a common need to exchange information in order to advance their missions

·         About NIEM-UML

NIEM-UML is a standard in progress for modeling NIEM in UML using a UML profile with Model Driven Architecture (MDA) provisioning capability to produce NIEM artifacts, such as an IEPD.

Getting started with NIEM-UML

NIEM-UML is a NIEM modeling tool which

·         fully implements the NIEM-UML specification,

·         provides enhanced modeling support for NIEM,

·         includes the full complement of NIEM Reference Models,

·         provides forward (model to MPD artifact) and reverse (MPD artifact to model) engineering for all MPD required artifacts,

·         provides upgrade of NIEM 2.0 to NIEM 2.1 packages,

·         provides comprehensive and NIEM-compliant model subsetting capabilities, including decomposition of substitutable elements

·         supports all MPD types and NIEM schema types,

·         comprehends and validates compliance to NDR (NIEM Design Rules), MPD (Model Package Description) specification, and other NIEM Specifications,

·         supports simplified constraint schema modeling and provisioning of functional constraint schema sets

·         supports business rules, including specification of constraints using OCL,

·         supports OCL parsing, validation, and execution based on binding of eclipse OCL with MagicDraw, including all OCL, QVT, and other operations, with support for XML Primitive types,

·         (internally) contains MOF-based models for OCL, QVT, XSD, WSDL, Schematron, NIEM MPD Change logs,  MPD Catalog, NIEM Want lists, etc.

·         enables modeling, generation, editing, browsing of UML Instance Models for Sample Documents, and provisioning  those models as XML Instance documents,

·         comprehension and provisioning of MPD catalog,

·         provisioning of MPD master-document,

·         provisioning of MPD change log, based on integration with Magic Draw model differencing engine,

·         provisioning of MPD want list,

·         active validation:

o   Compliance with NDR, MPD, and other NIEM rules/specifications

o   Compliance with all UML constraints (using the comprehensive eclipse based algorithms implementing OMG UML specified constraints)

o   Compliance with XSD constraints (using eclipse based XSD validation machinery) for schema models and corresponding Instance document models

o   Compliance with (eclipse) OCL syntax

o   Compliance of Instance models with OCL specified constraints (i.e., dynamic OCL execution)

·         Search engine which searches all NIEM Reference Models for similarities, whether loaded or not (based on Apache Lucene technology)

·         Tables for facilitating analysis and information entry

·         Support for EIEM development and use, including subsetted NIEM Reference Models to reduce exposed complexity

 

Below are listed some of the topics you might be interested when starting to work with NIEM-UML:
