Creating Classifiers from NIEM Tool Palette

All NIEM-UML relevant Classifiers may be created from the NIEM Tool Palette.   The Classifiers are grouped into the following categories, each corresponding to a pull-down list of tools:

·         Class.   Includes:

o   <<ObjectType>>

o   <<AdapterType>>

o   <<AssociationType>>

o   <<AugmentationType>>

o   <<MetadataType>>

o   <<Choice>>

o   Class

·         <<PropertyHolder>>  (not a list)

·         Datatype.  Includes:

o   <<ValueRestriction>>

o   <<List>>

o   <<Union>>

o   DataType

o   PrimitiveType

o   Enumeration

Procedure for creating a Classifier from NIEM Tool Palette:

·         Select one of the tools enumerated above

·         Move cursor to diagram open space

·         Click to add the type of Classifier to the <<InformationModel>> owning the diagram

Repetitive Classifier creation may be achieved in “sticky” mode using the following procedure:

·         Click “Sticky Button” icon located above the NIEM Tool Palette.

·         Select one of the tools enumerated above.

·         Move cursor to diagram open space.

·         Each click will add a new Classifier to the <<InformationModel>> owning the diagram.

 

 

Creating Classifiers from Generalization Tool

Many of the NIEM Classifiers may be created as part of a non-targeted Generalization relationship.  The functionality is available for the following tools:

·         Generalization from NIEM Tool Palette

·         Inverse Generalization from NIEM Tool Palette

·         Generalization Smart Manipulator in context of a Classifier

·         Inverse Generalization Smart Manipulator in context of a Classifier

 

When one of these tools targets open space on the diagram, a new Classifier will be created.  The new Classifier is of the same meta-class as the source Classifier, and will normally be Stereotyped with the same NIEM-UML Stereotype, subject to NIEM-UML, NDR, MPD, XSD, and UML constraints. 

 

 

Creating Classifiers from Context Menus

All NIEM-UML relevant Classifiers may be created from the context menus of qualifying elements selected in a browser or diagram.

 

To create Classifiers from a Diagram owned by an <<InformationModel>>:

·         Select the background.

·         Right click:  NIEMèadd *

Alternatively, from a Diagram containing <<InformationModel>> Packages:

·         Select an <<InformationModel>>.

·         Right click:  NIEMèadd *

 

 

To create Classifiers from a browser:

•      Select an <<InformationModel>>

•      right clickèNew ElementèNIEMè*

 

 

Creating Classifiers from a Selection Dialog

For Selection Dialogs which allow it, the following procedure may be used from the Selection Dialog to create any NIEM-UML relevant Classifiers:

•      Push Creation Mode button.  This will enable the Create action.

•      Select an editable <<InformationModel>> Package to be the owner of the new Classifier.

•      Push the Create button.  A pop-up menu lists the elements which may be created in the context of the selected <<InformationModel>> Package.

•      A new element is created when one of the menu items is selected.

•      Alternatively, the Clone action may be used to create Classifiers from existing Classifiers.

 

 

 

Populating the Classifier Specification

The <<ValueRestriction>> and <<XSDRepresentationRestriction>> have tag values corresponding to XML Schema Facets.   The values of the tags are set using the respective Specification Dialogs.  In either case:

•      Open the specification for either a <<ValueRestriction>> or a <<XSDRepresentationRestriction>> (an action available on the context menus for browsers or diagrams).

•      Select the Tags category from the tree on the left panel.

•      Scroll down, if necessary,  to the appropriate stereotype on the right panel.

•      Create/Remove/Edit Values for each tag as appropriate.
