Properties

Creating Properties from NIEM Tool Palette

NIEM Properties may be specified in conjunction with Associations.   When used in conjunction with Associations, the visualization of the Property on the Diagram may be more effective then when an Association is not used.  For NIEM, there is no real difference between a Property with an Association and a Property without an Association.  The Association related tools from the NIEM Tool Palette create Properties (which have Associations).   All the Association tools are grouped into a single category.  The tools include:

•      Association Class.  Creates a UML Association Class as a representation for a NIEM AssociationType.  In this case, there are two Properties created, both are owned by the Association Class, and each corresponds to an end of the Association.

•      <<RoleOf>>.    A <<RoleOf>> Property is owned by the client and has the Supplier as its type.   The tool creates a directed association to visualize the Property.

•      Reference.  The Property is represented as a directed, non-aggregate, association.

•      Content.  The Property is represented as a directed, aggregate, association.

•      Association.  There are two Properties created, one is owned by the client and the other is owned by the supplier.  The client property is typed by the supplier and the supplier property is typed by the client.  The tool creates a non-directed, non-aggregate, association to visualize the Property.

To create properties using the association tools:

•      Select an Association tool from the above list.

•      Click the client of the Association.

•      Drag the other end to the supplier of the Association.

 

 

 

Creating Properties from Smart Manipulators

NIEM Properties may be specified in conjunction with the Smart Manipulators of a Classifier.   These are similar to the Properties/Associations available in the NIEM Tool Palette, but there are a few additional cases.  The smart manipulators include:

•      Association.   This is like the Association tool.  There are two Properties created, one is owned by the client and the other is owned by the supplier.  The client property is typed by the supplier and the supplier property is typed by the client.  Neither side is aggregate.

•      Aggregation.  There are two Properties created, one is owned by the client and the other is owned by the supplier.  The client property is typed by the supplier and the supplier property is typed by the client.  The client property is aggregate, the supplier property is not aggregate.

•      Composition.  There are two Properties created, one is owned by the client and the other is owned by the supplier.  The client property is typed by the supplier and the supplier property is typed by the client.  The client property is composite, the supplier property is not aggregate.

•      Directed Association.  There is a property created for the client and it is not aggregate.  There is a non-navigable property owned by the Association which does not have any target NIEM artifact counterpart.

•      Directed Association.  There is a property created for the client and it is aggregate.  There is a non-navigable property owned by the Association which does not have any target NIEM artifact counterpart.

•      Directed Composition.  There is a property created for the client and it is composite.  There is a non-navigable property owned by the Association which does not have any target NIEM artifact counterpart.   There is no difference in the target NIEM artifacts between this association and a directed association.

 

To create properties using the smart manipulators:

•      Select a Classifier on the Diagram.   The Classifier is considered the client for this action.

•      Click the desired association Smart Manipulator.

•      Click another Classifier.  This Classifier is considered the supplier for this action.   Result is creation of association and properties, configured as enumerated above.

 

 

Creating Properties from Context Menus

The context menu for a selected Classifier in either a browser or a diagram provides for creating any kind of (contextually legal) NIEM “Property”.  These kinds of Properties will not be parts of an association.  The kinds of Properties include:

•      <<XSDAnyProperty>>

•      <<XSDProperty>>

•      <<SequenceID>>

•      Unstereotyped UML Property

In addition, there are convenience actions to add Properties:

•      Add multiple properties, each typed by a selected XML Primitive Type.

 

From a diagram, properties are created using the following procedure:

•      Select a Classifier

•      Right click: select one of the following actions:

o   Insert New Attribute.   Creates an unstereotyped UML Property.

o   Add <<XSDAnyProperty>>

o   Add <<XSDProperty>>

o   Add <<SequenceID>>

o   Add Primitive Properties.  This action opens a selection dialog enabling selection of one or more XML Primitive Types.  There will be one Property created for each selected type.

 

 

 

From a browser, properties are created using the following procedure:

•      Select a Classifier

•      Right click: select one of the following actions:

o   New ElementèProperty.   Creates an unstereotyped UML Property.

o   New ElementèNIEMè <<XSDAnyProperty>>

o   New ElementèNIEMè <<XSDProperty>>

o   New ElementèNIEMè <<SequenceID>>

o   NIEMèAdd Primitive Properties.  This action opens a selection dialog enabling selection of one or more XML Primitive Types.  There will be one Property created for each selected type.

 

 

 

 

Creating Properties from <<References>>

One of the use of <<References>> is to explicitly declare the top level element (in a <<PropertyHolder>>) which is referenced by a Classifier owned Property.  The top level element may be in the same or a different namespace as the referencing Classifier.  This use of <<References>> is between Properties.  The Properties at either end of the <<References>> must have the same NIEM Name and type.

In cases where it is desired to <<References>> a <<PropertyHolder>> Property, but the Property has not yet been defined in a <<PropertyHolder>>, the <<References>> tool may be used to both create the <<PropertyHolder>> Property and create the <<References>> to it.  This variant of the <<References>> tool behavior occurs when the client is a Property and the supplier is a <<PropertyHolder>>.

 

To create a Property in a <<PropertyHolder>> based on a client Property, use the following Procedure:

•      Select the <<References>> tool from the NIEM Tool Palette.

•      Select the (client) Property in some non-<<PropertyHolder>> Classifier.

•      Drag and drop on a <<PropertyHolder>> Class.

 

 

Alternatively, the same effect may be achieved via the <<References>> Smart Manipulator:

•      Select a Property in a non-<<PropertyHolder>> Classifier.

•      Select the <<References> Smart Manipulator.

•      Drag and drop on a <<PropertyHolder>> Class.

 

Sequencing Properties

The properties owned by a Classifier are ordered.  The order of the properties determines the sequence of elements in the provisioned schema types.  For subsetted models, the sequence of elements must be the same as the corresponding reference models.  For non-subsetted models, the properties may be ordered using the following procedure:

•      Open the Specification Dialog for a Classifier.  This action is available in the context menu for a Classifier from browsers or diagrams.

•      Scroll down to the “Owned Attribute” Property.

•      Select the “Owned Attribute” Property.

•      Click the button with the up/down arrows.  This will open the “Order Owned Attribute” Dialog.

 

 

•      In the “Order Owned Attribute” Dialog, select a Property and use the Up/Down push buttons to re-order the Property.
