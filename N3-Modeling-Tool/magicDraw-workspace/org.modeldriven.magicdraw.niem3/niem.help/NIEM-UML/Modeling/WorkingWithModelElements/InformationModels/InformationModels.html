Information Models

Creating an <<InformationModel>>

 

New <<InformationModel>>s are typically extension models. 

·         Reference models are part of NIEM Releases and are available as read-only, pre-defined, model libraries.

·         Subset models are created using NIEM subsetting actions.

·         Constraint models are created using NIEM constraint and subsetting actions.

·         Exchange models typically contain only top-level elements whose types are defined by an extension model.  An Exchange model is typically included in a new IEPD Project.

An <<InformationModel>> may be created using the following procedure:

·         Select a Package (not an <<InformationModel>>) from the containment browser.  Typically, this would be ..::XMLschemas::extension when creating an extension <<InformationModel>>.

·         Right click: New ElementèNIEMèInformationModel.   This will create a new Package stereotyped as <<InformationModel>>.

·         The new <<InformationModel>> appears in the containment browser with a prompt to enter name. 

·         Enter the name for the new <<InformationModel>>.

 

 

To create a diagram for the <<InformationModel>>:

•      Select the <<InformationModel>> in the containment browser.

•      Right click:  New DiagramèClass Diagram.  This will create a new Diagram in the Diagram area.  The Diagram element also appears in the containment browser with a prompt to enter name.

•      Enter name of diagram

 

An alternative procedure for creating <<InformationModel>>s:

·         Select a Package (not an <<InformationModel>> from the containment browser.  Typically this would be ..::XMLschemas::extension when creating an extension <<InformationModel>>.

·         Right click:  New DiagramèClass Diagram.  This will create a new Diagram in the Diagram area.  The Diagram element also appears in the containment browser with a prompt to enter name.

·         Enter name of diagram.

·         Within the NIEM diagram toolbar, select InformationModel and drag it to the diagram area.  A new <<InformationModel>> will be created as a nested Package of the package owning the diagram.

 

<<InformationModel>> Specification

The <<InformationModel>> has tags which determine how it, and its content, is to be validated and transformed to an MPD Schema.  These tags may be edited in the Specification Dialog using the following procedure:

•      Select the <<InformationModel>> in a browser or diagram.

•      Right click è Specification.   This will open a Specification Dialog for the <<InformationModel>>.

•      Within the Specification Dialog, Select “Tags” category in the tree on the left pane.

•      The following tags should be reviewed and assigned values as appropriate. 

•      defaultPurpose.   Click Create value push button and set value to extension.

•      defaultPrefix.  If a specific prefix is desired in generated MPD Schema, set its value here.  If no value is set, or there is a name conflict, a prefix will be assigned by the schema meta-model.

•      isConformant.   This should normally be set true to indicate that the MPD Schema is NIEM-conformant.

•      targetNamespace.  The value should be the unique target namespace for the MPD Schema.

•      Version. 

 

 

An alternate procedure may be followed to set some tag values:

·         Select the <<InformationModel>> in a browser or diagram.

·         Right click: select one of following:

o   isConformant checkbox.  Selecting this will toggle value of isConformant.

o   defaultPurpose.  Choose value from the list of choices.
