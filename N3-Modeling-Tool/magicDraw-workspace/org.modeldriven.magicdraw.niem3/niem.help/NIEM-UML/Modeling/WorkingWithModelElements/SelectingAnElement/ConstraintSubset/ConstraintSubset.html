<<InformationModel>> Selection for constraint subsetting

A Selection dialog used to select  <<InformationModel>>s to be constrained.   Constraint models are used to express constraints which cannot be otherwise expressed via subsetting in the base schema set for an MPD.    In particular, the constraint schema may constrain the type of Properties, which is not allowed in subset models.  The constraint models need only specify overrides to the base schema set.   When constraint schemas are provisioned, a set of fully populated constraint schemas are produced from the base schema set, then augmented with the override specifications specified in constraint <<InformationModel>>s.

 

·         The action to open this selection dialog is available as a contextual action on a browser or a diagram when the selected element is a non-<<InformationModel>> Package.

o   Right clickèNIEMèAdd Constraint Models.   This will open the Selection Dialog.

·         The selection choices include <<InformationModel>>s which are editable and are reference, subset, or extension models.

·         The selection choices do not include <<InformationModel>>s which are already being constrained.

·         Upon completion of the dialog, for each selected <<InformationModel>>:

o   A corresponding constraint <<InformationModel>> is created.

o   A <<References>> Realization is made from the constraint <<InformationModel>> to the constrained <<InformationModel>>.

·         When actioned in the context of a diagram, symbols for the new elements are added to the diagram.
